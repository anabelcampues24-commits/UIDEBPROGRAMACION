Introducción

El presente proyecto tiene como finalidad desarrollar el software denominado Juego del Ahorcado, utilizando el lenguaje de programación Python.

Este trabajo permite aplicar los conocimientos adquiridos en la asignatura de Lógica de Programación, tales como estructuras condicionales, ciclos repetitivos, variables, validaciones y control de flujo.


Objetivo General

Desarrollar un programa interactivo que simule el juego del ahorcado, aplicando estructuras básicas de programación.



1.	 Entorno de Programación
El entorno de programación comprende el conjunto de herramientas necesarias para la construcción, ejecución y prueba de programas informáticos. En el presente proyecto se empleó el lenguaje Python, seleccionado por su sintaxis clara, legibilidad y amplia aplicabilidad en el ámbito académico.
2.	Diagramas de Flujo
El diagrama de flujo es una representación gráfica del algoritmo mediante símbolos estandarizados que describen el flujo de ejecución del programa.
Los principales símbolos utilizados son:
•	Óvalo: Inicio y fin.
•	Rectángulo: Procesos.
•	Rombo: Decisiones.
•	Paralelogramo: Entrada y salida de datos.
La elaboración del diagrama de flujo facilita la comprensión visual del proceso lógico y permite detectar posibles inconsistencias antes de la codificación.

3.	 Manejo de Datos
El manejo de datos constituye un elemento fundamental en la programación, ya que permite almacenar, procesar y manipular información dentro del sistema.
En el desarrollo del Juego del Ahorcado se utilizaron los siguientes tipos de datos:
3.1.	 Variables
Las variables son espacios de memoria destinados al almacenamiento temporal de información. En el programa se emplearon variables para:
•	Almacenar la palabra secreta.
•	Controlar el número de intentos.
•	Registrar las letras ingresadas por el usuario.
3.2.	 Tipos de Datos
•	Cadenas de texto (String): Representan la palabra a adivinar.
•	Enteros (Integer): Controlan el número de intentos disponibles.
•	Listas (List): Permiten almacenar dinámicamente las letras ingresadas.
4.	Algoritmos y Diagramas de Flujo

4.1.	Algoritmos
Un algoritmo se define como una secuencia finita y ordenada de pasos lógicos que conducen a la solución de un problema. Debe cumplir con características fundamentales como precisión, finitud y determinismo.
       El algoritmo del Juego del Ahorcado contempla:
I.	Inicialización de variables.
II.	Evaluación de condición de continuidad.
III.	Solicitud de entrada al usuario.
IV.	Verificación de coincidencia.
V.	Actualización de estado del juego.
VI.	Presentación del resultado final.

5. Estructuras Condicionales
Las estructuras condicionales permiten que el programa tome decisiones en función del cumplimiento de una condición lógica. En el lenguaje Python se implementan mediante las instrucciones if, elif y else.
En el Juego del Ahorcado, las condicionales se utilizan para:
•	Determinar si la letra ingresada pertenece a la palabra secreta.
•	Verificar si el usuario ha agotado sus intentos.
•	Evaluar si la palabra ha sido completada.

6. Bucles o Estructuras Repetitivas
Los bucles permiten ejecutar un bloque de instrucciones de manera repetitiva mientras se cumpla una condición determinada.
En el proyecto se utilizó la estructura while, la cual mantiene activo el juego mientras existan intentos disponibles. Esta estructura posibilita la interacción continua entre el usuario y el sistema hasta que se alcance una condición de finalización.
El uso adecuado de bucles garantiza dinamismo y control en la ejecución del programa.


7. Estructuras de Datos
Las estructuras de datos permiten organizar y gestionar la información dentro del programa.
En el desarrollo del sistema se utilizó la estructura tipo lista para almacenar las letras ingresadas por el usuario, lo que permitió:
•	Evitar el ingreso repetido de caracteres.
•	Verificar coincidencias.
•	Mostrar el progreso parcial del jugador.
8. Funciones
Las funciones constituyen bloques de código reutilizables que permiten dividir un programa en módulos independientes. Su utilización favorece la modularidad, legibilidad y mantenimiento del software.
Aunque el Juego del Ahorcado puede implementarse de manera secuencial, la incorporación de funciones permite:
•	Separar la lógica de validación.
•	Encapsular la impresión del estado del juego.
•	Facilitar futuras mejoras o ampliaciones.
